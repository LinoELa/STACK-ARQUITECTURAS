# Carpeta MOBILE

Este archivo explica que representa la carpeta `MOBILE`, para que sirve y como se lee.

---

## 1. Que es esta carpeta

La carpeta `MOBILE` guarda referencias para arquitectura y organizacion de aplicaciones moviles.

Aqui se documentan dos cosas:

- estilos de arquitectura
- ejemplos por framework o plataforma

---

## 2. Que contiene

Esta carpeta contiene:

- `@mobile.md`
- `ARQUITECTURAS/`
- `FRAMEWORKS/`

### `ARQUITECTURAS/`

Guarda estilos de organizacion como:

- `Hexagonal`
- `Modular`

### `FRAMEWORKS/`

Guarda ejemplos o referencias mas concretas por tecnologia, por ejemplo:

- React Native
- Flutter
- Kotlin
- SwiftUI

---

## 3. Para que sirve

Esta carpeta sirve para:

- comparar formas de organizar una app movil
- elegir una base para un proyecto nuevo
- explicar la estructura del codigo antes de implementarlo
- usar estas guias como plantilla de referencia

---

## 4. Como leerla

La lectura recomendada es esta:

1. entrar en `ARQUITECTURAS/`
2. elegir `Hexagonal` o `Modular`
3. leer primero `@hexagonal.md` o `@modular.md`
4. despues comparar `MVP` y `Completo`

---

## 5. Resumen rapido

`MOBILE` es la carpeta de referencia para arquitectura movil.

Su idea principal es:

- separar estilos de arquitectura
- separar ejemplos por plataforma o framework
- dejar guias pequeñas y completas
- hacer visible como se organizaria una app movil real
