# Carpeta FRONTEND

Este archivo explica que representa la carpeta `FRONTEND`, para que sirve y como se lee.

---

## 1. Que es esta carpeta

La carpeta `FRONTEND` guarda referencias para arquitectura y organizacion de aplicaciones web.

Aqui se documentan dos cosas:

- estilos de arquitectura
- ejemplos por framework

---

## 2. Que contiene

Esta carpeta contiene:

- `@frontend.md`
- `ARQUITECTURAS/`
- `FRAMEWORKS/`

### `ARQUITECTURAS/`

Guarda estilos de organizacion como:

- `Hexagonal`
- `Modular`

### `FRAMEWORKS/`

Guarda ejemplos o referencias mas concretas por framework, por ejemplo:

- React
- Next.js

---

## 3. Para que sirve

Esta carpeta sirve para:

- comparar formas de organizar un frontend
- elegir una base para un proyecto nuevo
- explicar la estructura del codigo antes de implementarlo
- usar estas guias como plantilla de referencia

---

## 4. Como leerla

La lectura recomendada es esta:

1. entrar en `ARQUITECTURAS/`
2. elegir `Hexagonal` o `Modular`
3. leer primero `@hexagonal.md` o `@modular.md`
4. despues comparar `MVP` y `Completo`

---

## 5. Resumen rapido

`FRONTEND` es la carpeta de referencia para arquitectura web.

Su idea principal es:

- separar estilos de arquitectura
- separar ejemplos por framework
- dejar guias pequeñas y completas
- hacer visible como se organizaria un frontend real
