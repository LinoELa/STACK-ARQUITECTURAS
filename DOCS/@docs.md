# Carpeta DOCS

Este archivo explica que representa la carpeta `DOCS`, para que sirve y como se lee.

---

## 1. Que es esta carpeta

La carpeta `DOCS` guarda documentacion transversal del repositorio.

No esta pensada para un tipo de aplicacion concreto, sino para guias reutilizables.

---

## 2. Que contiene

Esta carpeta contiene:

- `@docs.md`
- `@hexagonal-VS-modular.md`
- `Architecture/`
- `Reference/`

### `Architecture/`

Guarda documentos de arquitectura organizados por estilo:

- `Hexagonal/`
- `Modular/`
- `@architecture.md`

### `Reference/`

Guarda referencias rapidas o apoyo documental como:

- flujo de GitHub
- estados HTTP
- checklist de preparacion

---

## 3. Para que sirve

Esta carpeta sirve para:

- centralizar documentacion reutilizable
- guardar reglas generales que no dependen de una sola app
- dejar referencias comunes para varios proyectos
- comparar estilos de arquitectura antes de elegir uno

---

## 4. Como leerla

La lectura recomendada es esta:

1. leer `@hexagonal-VS-modular.md` si quieres comparar enfoques
2. entrar en `Architecture/` si buscas estructura o reglas
3. entrar en `Reference/` si buscas apoyo rapido o checklist

---

## 5. Resumen rapido

`DOCS` es la carpeta comun de documentacion transversal del repositorio.

Su idea principal es:

- separar arquitectura general de referencias rapidas
- dejar documentacion reutilizable
- complementar las carpetas `BACKEND`, `FRONTEND` y `MOBILE`
- ayudar a elegir entre `Hexagonal` y `Modular`
